# 算法题

## 目录说明

这里存放算法题的正式沉淀内容。

按分类分目录存放，推荐结构如下：

- `<分类目录>/<题目目录>/01-question.js`
- `<分类目录>/<题目目录>/02-answer.js`

每道题使用独立目录，目录内固定包含：

- `01-question.js`
- `02-answer.js`

推荐分类目录：

- `01-数组与哈希表`
- `02-链表`
- `03-栈与队列`
- `04-二叉树与遍历`
- `05-双指针`
- `06-滑动窗口`
- `07-排序与查找`
- `08-递归与回溯`
- `09-贪心`
- `10-动态规划`

## 命名建议

- `01-数组与哈希表/001-两数之和`
- `04-二叉树与遍历/002-二叉树层序遍历`
- `06-滑动窗口/算法题-中级-滑动窗口最大值`

## 编写要求

- `01-question.js` 顶部注释写清楚题目、示例、作答要求、测试说明，并提供不含答案的答题区和执行区域
- `02-answer.js` 顶部注释写清楚解题思路、复杂度和复杂度推导，并提供最优解和执行区域
- 优先保留适合前端面试的题型和表达方式

## 默认出题约定

- 如果只说“出算法题”，默认按高频算法分类体系出题
- 默认分类顺序：
  `数组与哈希表`、`链表`、`栈与队列`、`二叉树与遍历`、`双指针`、`滑动窗口`、`排序与查找`、`递归与回溯`、`贪心`、`动态规划`
- 如果没有说明难度，默认出 `简单` 题
- 如果没有说明数量，默认出 `1` 题
- 如果没有明确要求生成文件，默认先在聊天中出题，不直接写入本目录
- 只有在明确要求“写进仓库”“生成文件”时，才按“分类目录 + 题目目录 + 双文件”的结构落地到本目录
