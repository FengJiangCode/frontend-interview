# 算法与手写题

## 学习目标

- 提升高频算法题的理解、拆解与表达能力
- 建立常见题型的解题模板与复习体系
- 同时覆盖算法题与前端高频手写题准备

## 目录结构

```text
7-算法与手写题/
├── README.md
├── 01-算法题/
│   └── <题目目录>/
│       ├── 01-question.js
│       └── 02-answer.js
└── 02-手写题/
    └── <题目目录>/
        ├── 01-question.js
        └── 02-answer.js
```

题目目录命名建议：

- 优先使用 `题号-题目名`
- 如果没有编号，使用 `类型-难度-题目名`

仓库内还提供了一个本地 skill：

- `.codex/skills/interview-question-generator`

后续如果你让我“出一道算法题”或“来一组手写题”，可以按这套结构直接沉淀到仓库里。默认执行方式为 `node 01-question.js` 和 `node 02-answer.js`。

## 建议整理内容

- 算法基础题型
- 数组
- 链表
- 栈与队列
- 哈希表
- 树
- 图
- 排序与搜索
- 双指针
- 滑动窗口
- 递归与回溯
- 贪心
- 动态规划
- 高频手写题
- 手写 `call` / `apply` / `bind`
- 手写 `new`
- 手写继承
- 手写深拷贝
- 手写 Promise
- 手写防抖与节流
- 手写发布订阅
- 手写数组去重
- 手写扁平化
- 手写柯里化
- 手写并发控制

## 面试怎么准备

- 算法题整理题目、思路、复杂度和易错点
- 手写题整理实现过程、边界情况和常见追问
- 区分“知道答案”“能独立写”“能边写边讲清楚”三种状态
