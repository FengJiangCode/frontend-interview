# 手写题

## 目录说明

这里存放前端高频手写题的正式沉淀内容。

手写题目录不按分类拆分，
统一直接按顺序编号沉淀。

每道题使用独立目录，目录内固定包含：

- `01-question.js`
- `02-answer.js`

## 命名建议

- `001-实现节流`
- `002-实现深拷贝`
- `003-实现 Promise.all`

## 编写要求

- `01-question.js` 顶部注释写清楚目标行为、示例和测试说明，并提供不含答案的答题区和执行区域
- `02-answer.js` 顶部注释写清楚解题思路、复杂度和边界情况，并提供最优解和执行区域
- 优先覆盖 JavaScript 面试高频实现题

## 默认出题约定

- 手写题不单独分分类目录，统一直接放在 `02-手写题` 根目录下，按顺序编号新增
- 新增题目前需要先检查当前已有最大编号，后续题目在现有编号基础上顺延，不重复占用旧编号
- 出题前需要先检查本目录下已有题目，尽量避免重复出已经存在的原题
- 如果只是同一考点下的变形题、进阶题或不同实现方式，可以继续出
- 默认不重复出仓库里已经有的完全同题题目，除非用户明确要求重复练习
