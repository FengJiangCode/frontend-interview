# AGENT.md

本仓库用于面试准备、知识复盘、项目练习与输出沉淀。

目标不是单纯堆资料，而是逐步形成一套可复用、可检索、可展示的个人学习资产。

## 仓库定位

- 记录面试相关知识点、题目、答案与思考
- 沉淀学习过程中的示例代码、实验和总结
- 维护一套适合长期复习的结构化资料
- 在需要时快速产出可读、可信、可复述的面试回答

## 工作原则

- 优先清晰，再追求全面
- 优先沉淀自己的理解，不机械搬运外部内容
- 每份内容尽量做到可复习、可搜索、可复述
- 小步提交，持续整理，避免一次性堆积大量未整理内容
- 结论要尽量有依据，涉及时效性内容时标注日期

## 内容组织建议

后续新增内容时，优先按主题拆分，而不是全部堆在根目录。

推荐目录结构：

```text
.
├── README.md
├── AGENT.md
├── notes/                # 学习笔记
├── interview/            # 面试题与标准回答
├── projects/             # 练习项目与 demo
├── algorithms/           # 算法与数据结构
├── system-design/        # 系统设计相关内容
├── resume/               # 简历素材、项目表述
└── assets/               # 图片、流程图、附件
```

如果当前阶段内容较少，可以先创建文档，再逐步拆目录，不要求一步到位。

## 文档编写规范

- 默认使用 Markdown
- 文件名尽量语义化，优先使用小写英文加连字符；如果内容主要面向中文学习，也可以使用清晰的中文命名
- 一篇文档只解决一个主题，避免内容过于发散
- 文档开头尽量包含以下信息
  - 主题
  - 创建日期
  - 最近更新日期
  - 标签
  - 当前状态：`草稿`、`整理中`、`已完成`
- 长文档优先包含目录或清晰分节

推荐文档结构：

```md
# 标题

## 背景

## 核心概念

## 常见问题

## 我的理解

## 示例 / 代码

## 易错点

## 面试怎么说

## 待补充
```

## 面试题内容规范

每道题建议尽量包含：

- 题目
- 简答版答案：适合 1 分钟内表达
- 展开版答案：适合深入追问
- 关键点列表
- 常见追问
- 示例或代码片段
- 容易答偏的地方

如果是八股题，不只写“标准答案”，还要补上“我会怎么说”，让内容更贴近真实表达。

## 代码与练习规范

- 示例代码以“最小可运行”为优先
- 不为了完整而引入过多无关依赖
- 每个练习建议配一个简短说明，写清楚目的、重点和结论
- 如果某个 demo 是为了面试讲项目，补充“亮点、取舍、问题复盘”

## 更新方式

每次新增或修改内容时，优先做到下面几件事：

- 补充 `README.md` 中的索引或说明
- 为新主题选择稳定的位置，不随意重复建文件
- 对旧内容进行轻量整理，避免仓库持续失序
- 如果结论发生变化，直接更新原文并注明更新时间

## README 建议职责

`README.md` 适合作为仓库首页导航，建议长期维护以下内容：

- 仓库目标
- 当前学习方向
- 目录索引
- 重点专题入口
- 最近更新记录

## Agent 协作要求

当 AI Agent 在本仓库中协助工作时，默认遵循以下规则：

- 优先帮助整理、归纳、补全，而不是制造冗余内容
- 保持现有结构的连续性，不随意改名或重组大量文件
- 新增文档时，尽量遵循本文件中的结构规范
- 输出内容优先中文，必要时保留英文术语
- 对不确定或有时效性的内容，应明确标注假设或日期
- 如果需要生成面试答案，优先区分“简答版”和“展开版”
- 如果需要生成学习笔记，优先补充“我的理解”“易错点”“如何在面试中表达”
- 避免生成空泛套话，尽量产出可直接复习的内容

## 当前阶段的默认策略

在仓库早期，优先级建议如下：

1. 完善 README，形成导航入口
2. 建立核心目录结构
3. 补充高频面试题与高价值知识点
4. 逐步加入代码练习、算法题与项目表达素材
5. 定期合并重复内容，保持资料整洁

## 判断标准

一份内容是否值得保留，可以用以下标准判断：

- 以后是否还会复习
- 是否能帮助回答真实面试问题
- 是否体现了自己的理解
- 是否比收藏一条外链更有价值

如果答案大多是否定的，就不必保留太多噪音内容。

## 一句话原则

这个仓库里的内容，最终都应该服务于两件事：

- 帮助自己真正学会
- 帮助自己在面试中讲清楚
