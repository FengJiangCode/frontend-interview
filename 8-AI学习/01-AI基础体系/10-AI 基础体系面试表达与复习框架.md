# 10-AI 基础体系面试表达与复习框架

## 先说结论

如果只记一句话，可以先记这个：

- 复习 AI 基础体系，不只是记定义
- 更重要的是形成“层级感、边界感、组合感、场景感”

也就是说，你最终不是要背一堆词，而是要能把这些词组织成一套清楚的表达框架。

## 1. 复习时最重要的四个维度

### 1. 层级感

要先分清：

- 哪个是总概念
- 哪个是能力基础
- 哪个是系统形态
- 哪个是实现方式

### 2. 边界感

要知道：

- 它解决什么问题
- 它不解决什么问题
- 它和相邻概念最容易混在哪

### 3. 组合感

要理解：

- 真实系统往往不是单一概念
- 很多能力会组合出现

比如：

- Agent + RAG
- Workflow + RAG
- Skill + Tool Use + MCP

### 4. 场景感

要能落到真实产品或工作场景：

- 企业问答助手
- 研发助手
- 排障助手
- 文档助手

## 2. 一套适合面试回答的表达顺序

如果面试官问一个 AI 概念题，你可以优先按这个顺序答：

1. 先说它是什么
2. 再说它解决什么问题
3. 再说它和相邻概念的区别
4. 再说真实场景怎么用
5. 最后说边界和误区

这个顺序很稳，因为既有定义，又有判断力。

## 3. 一个通用回答模板

你可以把很多 AI 概念题都套进这个模板：

```text
它本质上是……
它主要解决的是……
它和……的区别在于……
真实系统里通常会和……组合使用
如果场景是……，它会更合适；如果场景是……，就不一定需要它
```

这个模板适合回答：

- RAG
- Agent
- Workflow
- Skill
- MCP
- Tool Use

## 4. 一个更适合复习的知识组织方式

可以把你已经写的专题分成四组来复习：

### 第一组：整体地图

- AI、大模型、Agent 的基本关系

### 第二组：系统形态

- RAG、Workflow、Agent 的区别
- Agent 的规划、执行与反思

### 第三组：系统运行机制

- Prompt、Context、Memory、Tool Use
- 记忆系统
- Tool Use 和 MCP 的关系

### 第四组：实现与误区

- RAG 的实现链路
- 常见误区总表

## 5. 面试里最常见的失分点

### 1. 只会背定义

只背一句“RAG 是检索增强生成”是不够的，面试官很容易接着追问。

### 2. 只讲优点，不讲边界

如果你说 Agent 很强，但说不出什么时候不该用，就会显得不够稳。

### 3. 不会落真实场景

如果不能结合企业问答助手、研发助手、排障场景来讲，就会显得很空。

## 6. 一分钟表达示例

如果面试官问你“你怎么理解现在常见的 AI 系统”，可以这样答：

```text
我通常会先把概念分层来看。
AI 是总概念，大模型是当前很重要的一类能力基础，Agent 则是建立在大模型上的任务执行系统。
在具体系统里，RAG 主要解决知识来源问题，Workflow 解决固定流程编排，Agent 解决动态决策问题。
而在运行机制上，Prompt、Context、Memory、Tool Use 决定系统到底能不能稳定完成任务。
如果再往实现层看，像 MCP 更偏外部能力的标准化接入，Tool Use 更偏实际调用行为。
我觉得把层级、边界和组合关系分清楚，理解起来会比较稳。
```

## 7. 复习时建议重点自测

- 你能不能用 1 分钟讲清这套基础体系
- 你能不能分别从“层级、边界、组合、场景”四个角度组织答案
- 如果面试官随机问一个概念，你能不能自然套用表达模板
