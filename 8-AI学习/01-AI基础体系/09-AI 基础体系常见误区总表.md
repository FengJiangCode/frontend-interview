# 09-AI 基础体系常见误区总表

## 先说结论

如果只记一句话，可以先记这个：

- 很多 AI 基础概念不是难在定义
- 而是难在边界

真正容易出错的，往往不是“不知道这个词”，而是“把两个词当成一回事”。

## 1. 常见误区：把 AI 等同于大模型

- 错误理解：AI 就是大模型
- 更准确的说法：大模型只是 AI 里的一个重要方向，AI 远比它范围大

## 2. 常见误区：把大模型等同于 Agent

- 错误理解：用了大模型就是 Agent
- 更准确的说法：Agent 是建立在大模型之上的系统形态，不是模型本身

## 3. 常见误区：把 RAG 当成完整系统

- 错误理解：做了 RAG 就等于做完了 AI 应用
- 更准确的说法：RAG 主要解决知识注入问题，不等于整个任务执行系统

## 4. 常见误区：把所有多步骤流程都叫 Agent

- 错误理解：只要有多个步骤就是 Agent
- 更准确的说法：如果步骤是固定预设的，更像 Workflow；只有需要动态决策时才更像 Agent

## 5. 常见误区：把 Skill 当成一段 Prompt

- 错误理解：Skill 不就是 prompt 模板吗
- 更准确的说法：Skill 更像稳定能力封装，通常还包括适用场景、步骤、输出格式和边界

## 6. 常见误区：把 MCP 当成插件市场

- 错误理解：MCP 就是一个插件列表
- 更准确的说法：MCP 更像标准化连接方式，重点是统一接入外部资源和工具

## 7. 常见误区：把 Tool Use 和 MCP 当成一个概念

- 错误理解：能调工具就是 MCP
- 更准确的说法：Tool Use 是调用行为，MCP 是能力接入方式

## 8. 常见误区：把 Prompt、Context、Memory 混在一起

- 错误理解：它们都是“喂给模型的文本”
- 更准确的说法：
  - Prompt 偏任务指令
  - Context 偏当前推理看到的全部信息
  - Memory 偏系统保存和复用历史信息的机制

## 9. 常见误区：觉得有了 Agent 就一定更高级

- 错误理解：Agent 一定比 Workflow 或 RAG 好
- 更准确的说法：是否需要 Agent，要看任务是否真的需要动态决策

## 10. 面试怎么说

如果面试里被追问“最容易混淆的点是什么”，可以这样回答：

```text
我觉得这些概念最容易混淆的地方不在定义，而在层级和边界。
比如 AI 和大模型不是一个层级，RAG 和 Agent 解决的问题也不一样，Tool Use 和 MCP 一个偏使用、一个偏接入。
如果把层级分清楚，很多概念其实会顺很多。
```

## 11. 复习时建议重点自测

- 你最容易混淆的三组概念是哪三组
- 你能不能分别用一句话区分它们
- 如果别人把它们混着说，你能不能温和地纠正
