# AI 基础体系学习清单

## 1. AI、大模型、Agent 的基本关系

- 延伸文档：[04-AI、大模型、Agent 的基本关系.md](./04-AI、大模型、Agent%20的基本关系.md)
- AI 是更大的概念，大模型是当前常见能力基础，Agent 是在模型之上加入目标、状态、工具和流程的应用形态
- 理解“大模型负责生成与推理”“Agent 负责协作与执行”这个基本分工
- 区分“能回答问题”和“能完成任务”之间的差异
- 面试或交流时，能用一两句话讲清三者关系

## 2. Prompt、Context、Memory、Tool Use

- 延伸文档：[03-Prompt、Context、Memory、Tool Use.md](./03-Prompt、Context、Memory、Tool%20Use.md)
- Prompt 是任务指令，Context 是当前上下文，Memory 是可持续状态，Tool Use 是能力扩展方式
- 理解为什么只靠 Prompt 很难完成复杂任务
- 理解 Context 和 Memory 的差异，避免把临时输入和长期状态混为一谈
- 理解 Tool Use 的作用：让模型具备查询、执行、读取外部信息的能力

## 3. RAG 是什么，解决什么问题

- 延伸文档：[05-RAG 的实现链路.md](./05-RAG%20的实现链路.md)
- 理解 RAG 的核心目标是“把外部知识在生成前引入上下文”
- 明确 RAG 适合知识问答、资料检索增强，不适合解决所有推理和执行问题
- 区分 RAG、微调、单纯长上下文三种常见方案
- 能说明 RAG 的价值、局限和常见误区

## 4. Workflow 与 Agent 的区别

- 延伸文档：[02-RAG、Workflow、Agent 的区别.md](./02-RAG、Workflow、Agent%20的区别.md)
- Workflow 更强调预定义流程和稳定编排
- Agent 更强调根据目标动态决策、调用工具和调整步骤
- 理解什么时候用固定流程就够了，什么时候需要更强的自主能力
- 避免把所有多步骤流程都叫 Agent

## 5. Skill 是什么

- Skill 可以理解为对特定任务能力、步骤和约束的封装
- Skill 的价值在于复用、稳定输出和减少重复描述
- 理解 Skill 和 Prompt 模板、工具调用之间的关系
- 能结合真实例子说明 Skill 适合解决哪类问题

## 6. MCP 是什么，为什么需要 MCP

- 延伸文档：[06-Tool Use 和 MCP 的关系.md](./06-Tool%20Use%20和%20MCP%20的关系.md)
- 理解 MCP 的价值是把模型和外部能力、数据源、工具访问方式做标准化连接
- 明确 MCP 不只是“插件目录”，而是让模型以统一方式接入外部上下文和能力
- 能说明 MCP 在生态中的位置：连接模型、Agent、工具、资源
- 理解为什么在多工具、多数据源场景下需要统一协议

## 7. Agent、Skill、MCP 的关系与协作

- 延伸文档：[01-Agent、Skill、MCP 关系梳理.md](./01-Agent、Skill、MCP%20关系梳理.md)
- Agent 负责面向目标组织任务
- Skill 负责沉淀特定任务的稳定能力
- MCP 负责把外部资源和工具能力规范化暴露出来
- 能用一个真实工作流例子讲清三者如何配合完成任务

## 8. 常见误区

- 延伸文档：[09-AI 基础体系常见误区总表.md](./09-AI%20基础体系常见误区总表.md)
- 把 Agent 当自动万能体，忽视边界和失败场景
- 把 RAG 当知识库万能解，忽视召回、排序和上下文质量
- 把 MCP 只理解成插件列表，忽视它的协议和连接价值
- 把 Skill 当作随便一段 Prompt，而没有考虑复用和约束
