# 08-Agent 的规划、执行与反思

## 先说结论

如果只记一句话，可以先记这个：

- 规划解决“先做什么、后做什么”
- 执行解决“真正把步骤做出来”
- 反思解决“发现问题后怎么修正”

一个像样的 Agent，通常不是一次生成到底，而是在这三者之间不断循环。

## 1. 为什么 Agent 需要这三步

如果只有规划，没有执行，系统只是纸上谈兵。  
如果只有执行，没有规划，系统容易乱做。  
如果没有反思，系统一旦走偏就会一直错下去。

所以这三步共同决定了 Agent 是否真的像“在完成任务”。

## 2. 规划是什么

规划可以理解为：

- 理解目标
- 拆分步骤
- 判断先后顺序
- 决定哪些地方要用工具

规划的核心不是把所有细节一次写死，而是给任务找一个合理的起点和路径。

## 3. 执行是什么

执行就是：

- 读资源
- 调工具
- 产出中间结果
- 把步骤真的跑起来

很多看起来像“智能”的行为，其实是执行阶段在发挥作用。

## 4. 反思是什么

反思可以理解为：

- 当前结果靠谱吗
- 有没有证据不足
- 前一步是不是走错了
- 要不要换路径、补一步、重试一次

它不是哲学意义上的反思，而是任务执行中的纠偏机制。

## 5. 一个最常见的循环

Agent 很多时候可以被理解成：

```text
规划 -> 执行 -> 检查 -> 调整 -> 再执行
```

这也是为什么 Agent 和固定 Workflow 不一样。  
Workflow 更像按既定步骤走，Agent 更强调中途动态调整。

## 6. 工作中的真实例子

### 例子 1：Bug 排查

- 规划：先看日志，再看代码，再看接口文档
- 执行：读取日志、搜索关键字、打开源码文件
- 反思：发现日志证据不足，于是补查缓存策略或历史变更

### 例子 2：知识助手

- 规划：先判断问题属于哪类知识域
- 执行：检索制度文档、提取相关片段、生成答案
- 反思：发现召回片段不够准，于是改写 query 再检索一次

## 7. 常见误区

### 1. 把规划理解成一次性固定计划

真实 Agent 的规划往往会在执行中被调整。

### 2. 把执行理解成模型自己“脑补”

执行通常需要真实资源、工具和外部数据支持。

### 3. 忽略反思

没有反思的 Agent，很容易把错误一路放大。

## 8. 面试怎么说

如果面试官问你“Agent 为什么不像普通问答”，可以这样回答：

```text
因为 Agent 通常不是一次性回答完，而是会先规划，再执行，再根据执行结果做反思和调整。
这让它更适合目标明确但路径不固定的任务，比如排障、代码分析或多步信息处理。
```

## 9. 复习时建议重点自测

- 规划和 Workflow 的固定编排有什么不同
- 为什么执行阶段必须依赖外部能力
- 为什么反思会直接影响 Agent 稳定性
- Bug 排查场景里，反思最可能发生在哪一步
