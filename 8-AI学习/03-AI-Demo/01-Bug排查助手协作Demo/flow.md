# Bug排查助手协作流程

```text
用户提交排查请求
  ->
Agent 接收目标
  ->
LLM 判断下一步应该调用什么能力
  ->
Agent 调用 Bug分析Skill
  ->
Skill 通过 MCP 读取日志、源码、接口文档
  ->
Skill 输出结构化证据
  ->
LLM 根据证据组织最终结论
  ->
Agent 输出定位、原因、证据、修复建议
```

## 这版流程为什么更贴近真实使用

- Agent 不直接“知道答案”，而是先调度
- LLM 不直接替代整个系统，而是承担决策与总结
- Skill 不只是文本模板，而是稳定任务能力
- MCP 不只是概念说明，而是被实际调用来读资源

## 角色分工

- Agent：流程组织者
- LLM：判断与总结
- Skill：方法封装
- MCP：资源连接层
