# 面试题生成器

这个 skill 用于生成两类面试练习题：

- 算法题
- 前端手写题

当用户提出类似下面的请求时，应使用这个 skill：

- “给我出一道算法题”
- “来一道 JS 手写题”
- “按中级难度来 5 道题”
- “生成可执行的题目文件”

## 默认行为

- 用户没有说明数量时，默认生成单题。
- 用户明确要求“多道题”“一套题”“一组题”时，生成题单。
- 用户没有说明难度时，默认使用 `中级`。
- 用户明确说明 `算法题` 时，只生成算法题。
- 用户明确说明 `手写题` 时，只生成手写题。
- 用户没有说明题型时，优先根据上下文判断。
- 如果上下文不足，默认生成算法题。

## 输出要求

每道题都应遵循仓库中的双文件结构，文件命名固定为：

- `01-question.js`
- `02-answer.js`

`01-question.js` 用于帮助用户理解题意和直接答题，要求：

- 顶部注释中包含题目标题、题目类型、难度、题目描述、举例说明、作答要求、测试说明
- 提供明确的 `TODO` 答题区域
- 不包含具体实现，具体实现必须由用户自行完成
- 使用 ES Modules 语法
- 提供带具体 case 的执行区域，方便直接运行并验证结果是否符合预期
- 可以直接通过 `node 01-question.js` 执行

`02-answer.js` 用于帮助用户理解最优解，要求：

- 顶部注释中包含题目标题、题目类型、难度、解题思路、时间复杂度、空间复杂度、复杂度如何计算、关键边界情况、面试追问
- 提供最优解实现
- 使用 ES Modules 语法
- 提供简单的执行区域，方便直接运行并查看结果
- 可以直接通过 `node 02-answer.js` 执行

输出内容应优先服务面试训练，不要直接照搬 LeetCode 原题描述。

## 仓库落地结构

当用户希望把题目直接写入仓库时，使用以下路径：

- `7-算法与手写题/01-算法题/<题目目录>/01-question.js`
- `7-算法与手写题/01-算法题/<题目目录>/02-answer.js`
- `7-算法与手写题/02-手写题/<题目目录>/01-question.js`
- `7-算法与手写题/02-手写题/<题目目录>/02-answer.js`

题目目录命名规则：

- 优先使用 `题号-题目名`
- 如果没有编号，使用 `类型-难度-题目名`

示例：

- `001-两数之和`
- `002-实现防抖`
- `算法题-中级-二叉树层序遍历`
- `手写题-初级-实现柯里化`

## 如何判断出题模式

- 出现 `出一道` `来一道` `练一道` 时，按单题模式处理。
- 出现 `来 5 题` `来一套` `给我一组` 时，按题单模式处理。
- 如果用户提到 `模拟面试`，优先使用面试风格出题：
  - 约束更清晰
  - 留出继续追问的空间
  - 示例简洁但贴近真实面试

## 模板文件

在需要将内容写入仓库时，优先复用以下静态 JS 模板：

- `assets/01-question.template.js`
- `assets/02-answer.template.js`

使用方式：

- 复制模板内容
- 替换题目标题、题目类型、难度、题目描述和示例
- 根据具体题目修改 `runExample()` 中的 case
