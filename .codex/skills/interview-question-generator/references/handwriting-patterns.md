# 手写题出题参考

## 适用范围

用于生成前端面试常见手写实现题，强调 JavaScript 基础、边界处理、实现过程表达与可执行验证。

## 常见题型

- 手写 `call` / `apply` / `bind`
- 手写 `new`
- 手写继承
- 手写深拷贝
- 手写 Promise
- 手写 `Promise.all`
- 手写防抖与节流
- 手写发布订阅
- 手写数组扁平化
- 手写柯里化
- 手写并发控制

## 出题风格

- 明确实现目标，不要求无意义的超长代码
- 写清楚输入、输出和行为约束
- 给出最少但有效的示例
- 鼓励追问异常情况、兼容性和边界处理

## 可执行校验建议

- `01-question.js` 中应让用户直接在答题区补完实现
- `01-question.js` 不要提前给出参考实现
- `01-question.js` 中保留简单调用示例，方便快速观察运行结果
- `02-answer.js` 中保留最优解与执行示例，方便对照验证
