# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 仓库定位

这是一个前端面试复习仓库，目标是沉淀可复习、可检索、可复述的个人知识资产。内容应服务于两件事：帮助真正学会，以及帮助在面试中讲清楚。

## 运行练习题

无构建工具、无测试框架、无编译流程。所有练习文件是纯 ES 模块，直接用 Node.js 运行：

```bash
node 01-question.js   # 练习题（含 TODO，需自己补全）
node 02-answer.js     # 参考答案（含复杂度分析）
```

## 仓库结构

9 个主题目录覆盖完整的面试范围：

| 目录 | 内容 |
|---|---|
| `1-HTML与CSS/` | HTML & CSS |
| `2-JavaScript核心/` | 作用域、闭包、原型链、Promise、事件循环 |
| `3-浏览器原理/` | URL→渲染、HTTP、缓存、跨域 |
| `4-框架能力/` | React / Vue 核心原理、Hooks、响应式 |
| `5-工程化/` | Webpack、Vite、Babel、CI/CD |
| `6-性能与安全/` | 性能指标、XSS、CSRF、CSP |
| `7-算法与手写题/` | 算法题 + 前端手写题 |
| `8-AI学习/` | AI 基础认知与职场应用 |
| `9-项目表达/` | 项目背景、难点、方案、结果的表达 |

### 算法与手写题（`7-算法与手写题/`）

两种类型：
- `01-算法题/` — 10 个分类子目录（数组/哈希、链表、栈与队列、二叉树、双指针、滑动窗口、排序/查找、递归/回溯、贪心、动态规划）
- `02-手写题/` — 顺序编号目录（`001-实现Promise.all/`、`006-实现柯里化/` 等）

每道题是一个目录，固定包含 `01-question.js`（待补全的 TODO 框架）和 `02-answer.js`（参考实现）。

## 内容规范

### 语言
默认使用**中文**输出内容，英文保留技术术语（如 `Promise`、`Proxy`、`BigInt`）。

### 面试题格式
每道题应包含：
- **简答版**：适合 1 分钟内表达
- **展开版**：适合深入追问
- 关键点列表
- 常见追问与答案
- 最小可运行示例代码
- 容易答偏的地方

### 文档元信息头部
新文档开头建议包含：
```
# 标题
- 创建日期: YYYY-MM-DD
- 最后更新: YYYY-MM-DD
- 标签: [tag1, tag2]
- 状态: 草稿 | 整理中 | 已完成
```

### 代码示例
保持最小可运行，不引入无关依赖。练习文件底部应包含内置测试用例（自执行 async IIFE）。

## Agent 协作规则

- 优先整理、归纳、补全现有内容，不制造冗余
- 保持现有目录结构，不随意改名或重组文件
- 时效性内容（浏览器 API、框架版本、规范）必须标注日期
- 生成面试答案时，始终区分"简答版"和"展开版"
- 避免空泛套话，产出可直接复习的内容
- 新增内容后，更新对应主题的 `README.md` 索引
